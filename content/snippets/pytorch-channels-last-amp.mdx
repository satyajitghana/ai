For conv-heavy models on NVIDIA tensor cores, two cheap changes compound: move the
model and inputs to `channels_last` memory format, and wrap the forward pass in
`autocast` with a `GradScaler`. The memory format keeps activations in a layout the
tensor cores prefer; AMP runs the math in bf16/fp16 while keeping a fp32 master copy.

```python
model = model.to(device, memory_format=torch.channels_last)
scaler = torch.amp.GradScaler("cuda")

for x, y in loader:
    x = x.to(device, memory_format=torch.channels_last, non_blocking=True)
    y = y.to(device, non_blocking=True)

    optimizer.zero_grad(set_to_none=True)
    with torch.autocast("cuda", dtype=torch.float16):
        loss = criterion(model(x), y)

    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```
