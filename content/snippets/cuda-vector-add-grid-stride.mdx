A grid-stride loop decouples the launch configuration from the problem size: each
thread processes multiple elements, striding by the total number of threads. This
keeps the kernel correct for any `n`, plays nicely with occupancy tuning, and lets
you reuse the same launch geometry across input sizes.

```cuda
__global__ void vadd(const float *a, const float *b, float *c, int n) {
  int stride = blockDim.x * gridDim.x;
  for (int i = blockIdx.x * blockDim.x + threadIdx.x; i < n; i += stride) {
    c[i] = a[i] + b[i];
  }
}

// Launch: size the grid to the device, not to n.
int block = 256;
int grid = (n + block - 1) / block;        // cap this for very large n
vadd<<<grid, block>>>(d_a, d_b, d_c, n);
```
